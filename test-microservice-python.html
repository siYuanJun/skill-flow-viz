Test Microservice Python

为微服务生成完整 Python 接口测试套件：业务流程集成测试 + 单接口参数校验。venv 隔离依赖、命令行可视化、致命错误即停、独立日志文件。需《接口全貌.md》前置。

执行全流程

pre 前置校验
输入
微服务名/端口/BASE_URL、接口文档（接口全貌.md）、输出目录、认证方式。
关卡
必须提供接口全貌.md；仅 dev 环境；必须用 python3 venv。
01 建 venv + 装依赖
动作
创建虚拟环境（非系统全局 pip），仅装 requests。
02 生成脚本骨架
产出
config.py / logger.py / main.py / run_all.sh / run_p0.sh / utils（http_client、data_extractor）。
03 业务流程测试（集成）
动作
真实 HTTP 请求模拟完整业务流程，参数从前置接口返回值提取。
认证
启动时登录获取 Cookie/Session，复用同一 Session，Token 过期自动重登。
04 参数校验测试（单元）
动作
每接口覆盖：必填缺失 / 类型错误 / 边界值 / 不存在 ID。
05 运行 + 致命即停
规则
FATAL（前置失败）→ sys.exit(1)；ERROR（单查询失败）→ 记录继续。不用 pytest。
产出
独立日志 logs/test_{时间戳}.log（与终端一致）。
06 数据清理
动作
按创建逆序通过 HTTP 清理，不直接操作数据库。

关键信息

触发条件

  • 微服务上线前接口全量回归测试
  • 业务流程通畅性验证
  • 接口参数校验自动化 / CI/CD 流水线测试

排除边界

  • Docker 环境测试（用 test-microservice-docker）
  • 性能 / 压力测试
  • 直接操作数据库
  • 无接口全貌.md 时不适用

产出物

  • 测试脚本目录（config / logger / main / scene_*.py / utils）
  • README.md 测试说明 + logs/test_*.log
