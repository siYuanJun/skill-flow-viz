执行全流程（需求门控 → 8 步生成）

00 第一关 · 文本质量扫描（不看项目）
动作
仅读需求文本，评估「可读性 + 完整度」。🟢 放行 → Step1；🟡 引导追问；🔴 不可解析直接驳回。
原则
零成本初筛，不评项目关联度 / 可行性（还没读项目）。
01 路径自动定位
动作
所有输入输出统一至 .ai-workflow/，无需询问用户。
02 接收需求
动作
从 .ai-workflow/meta-agent-collaboration/docs/ 或用户指定路径读取需求文档。
03 依赖检查 + 机械臂扫描
动作
校验必需文件，build_index + scan_schema 扫描项目全貌。
04 需求理解 + 第二关上下文深评
动作
评估「项目关联度 + 可行性」：🟢 放行 → 三层提取；🟠 警告（附证据，需确认）；🔴 驳回。三层模型：事实锚定→业务解读→技术推导。
05 模块划分
动作
按功能域 / 角色 / 技术维度聚合模块。
06 文档生成
动作
逐模块填充分析，技术落地点经机械臂反推验证（find_callers / scan_imports / scan_schema）。
07 质量校验
动作
全量覆盖、编号一致、原文保真、验证状态标注（✅/⚠️/❌）。
08 输出交付
产出
.ai-workflow/meta-agent-collaboration/docs/模块{N}-需求文档.md + 门控评估报告。

关键信息

排除边界

  • 一次性简单需求问答
