研发层 · 产品文档基座

产品 PRD 基座工程 ai-project-prompt · V1.0.0

从一个产品想法，交互式产出三层产品文档基座：产品PRD → 技术架构 → 数据模型概要。每层由专属「产品意识大脑」驱动——先溶解理解、再判断、再产出，每步显式确认是否继续，并按产品形态自适应文档结构。纯提示词技能（无 Python 脚本），含 evals 三件套 + yao 自检门控。

三层文档引擎（一眼看懂）

从产品想法到三层文档基座，七节点闭环。琥珀色为「确认门控」节点——AI 不擅自一口气全产出，必须等你确认才进入下一层。下方时间轴为逐步详情。

100%
滚轮缩放 · 拖拽平移 · 双击放大

核心设计：不是模板生成器，是产品意识大脑

三个大脑结构一致：溶解理解 → 做判断 → 产出文档，每个判断都明确、具体、有理由。判断驱动文档，而非模板填空。

✗ 模板填充式

直接套格式写文档，不理解就输出；判断模糊、P0/P1/P2 无理由、技术选型无 trade-off、实体归属含糊。

✓ 判断驱动式（本 Skill）

每步先展示判断（价值闭环 / 优先级划分 / 本期不做什么 / 不确定项），确认后再产出；判断明确、清晰、具体、可落地。

产品思维引擎

references/product-prd-brain.md
  1. 溶解：谁的什么问题 · 价值闭环 · 核心实体 · 形态约束
  2. 识别骨架：核心价值 / 支撑 / 增强 → P0/P1/P2
  3. 做判断：去掉它主闭环走不通=P0；每「不做」必须有理由
  4. 识别状态边界 · 产出 [产品名]-PRD.md

技术思维引擎

references/tech-architecture-brain.md
  1. 溶解：形态定层数 · 团队背景是硬约束 · 已有系统是锚点
  2. 做选型判断：为什么选 · 代价是什么 · 换的代价多大
  3. 高代价决策（如数据库）需更充分理由
  4. 产出 [产品名]-技术架构.md

数据思维引擎

references/data-model-brain.md
  1. 溶解：从 PRD 提取流动实体 · 按业务域分组
  2. 判定归属：谁写入 · 谁读取 · 存储形态（关系/NoSQL/事件流）
  3. 识别状态流转、归属模糊地带
  4. 产出 [产品名]-数据模型概要.md（不含 DDL）

执行全流程（7 步交互式 SOP）

01 收集产品信息（交互入口）
交互
触发后先收集：产品名称（用于命名）/ 项目代号 / 产品形态（Web·App·CLI·API·SDK·小程序·桌面·对话式·游戏·IoT）/ 一句话定位 / 目标用户 / 核心功能想法（3-8 项）/ 系统拓扑 / 已知约束。缺失的主动提 2-3 个关键问题，不猜。
02 产出产品 PRD（先判断，再产出）
加载
references/product-prd-brain.md 产品思维引擎。
先展示判断
核心价值闭环是什么？P0/P1/P2 划分逻辑？本期不做什么及理由？不确定什么需你补充？
产出
[产品名]-PRD.md，开头标注产品名 / 代号 / 形态 / 版本 / 日期。
03 确认门控 ⏸ 是否继续
门控
产出 PRD 后必须询问是否继续产出技术架构。确认 → 进入 04；说不 → 结束（你可之后单独触发架构产出）。
04 产出技术架构（先判断，再产出）
加载
references/tech-architecture-brain.md 技术思维引擎。
先展示判断
产品有几层技术栈？最关键的 3 个技术决策及理由？最大技术风险是什么？
产出
[产品名]-技术架构.md。
05 确认门控 ⏸ 是否继续
门控
产出架构后必须询问是否继续产出数据模型概要。确认 → 进入 06；说不 → 结束。
06 产出数据模型概要（先洞察，再产出）
加载
references/data-model-brain.md 数据思维引擎。
先展示洞察
识别出多少业务域 / 核心实体？哪些实体有状态流转（几个状态）？数据归属有无模糊地带？
产出
[产品名]-数据模型概要.md（不含 DDL）。
07 总结交叉校验
校验
PRD 功能模块 → 技术架构是否都有对应目录？PRD 业务实体 → 数据模型是否覆盖完整？数据模型状态枚举 → PRD 状态机是否一致？告知你校验结果。
单独产出
也可只说「帮我产出技术架构/数据模型」——先确认有无现成 PRD 可参考，有则基于 PRD，无则先收集信息。

关键信息

触发条件

  • 从产品想法搭建三层文档基座（PRD / 技术架构 / 数据模型）
  • 单独产出其中某一层
  • 产品形态自适应的文档结构需求

排除边界（Out of scope）

  • 单个功能 PRD（不涉及完整产品）
  • 纯代码审查 / 已有文档润色
  • 把框架源码蒸馏成开发契约 → feature-distiller
  • 画代码调用关系图谱 → graphify

产出物

  • [产品名]-PRD.md（做什么 + 为什么）
  • [产品名]-技术架构.md（怎么做）
  • [产品名]-数据模型概要.md（数据长什么样 + 归谁管，不含 DDL）

产品形态自适应

WebAppCLI APISDK小程序 桌面对话式游戏IoT

CLI 不写页面结构，SDK 不写用户角色，API 不写 UI 组件——按形态与使用场景落地文档。
