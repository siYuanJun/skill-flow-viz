Debug Frontend Param

前端参数类错误排查元提示词：不直接修后端，而是生成一份给「前端 agent 执行用」的标准化排查文档（背景 + 可执行 grep/curl + 结果汇总）。覆盖 4 类场景。

执行全流程（元提示词 → 排查文档）

01 提取输入意图
动作
从用户自然语言提取：问题类型（报错/咨询/痕迹排查）、关键信息（接口路径/参数名/报错内容）、排查方向。
02 选场景策略（4 类）
场景1
接口报错（后端已确认OK）→ 搜错误值硬编码 → 搜调用链路 → 搜上游数据源
场景2
数据源痕迹 → 搜独有接口路径 → 搜模块标识 → 搜特有 URL 格式
场景3
接口咨询 → 搜接口路径调用 → 搜封装函数 → 搜 import 引用
场景4
通用确认 → 聚焦可 grep 关键词，无法 grep 的用自然语言提问
03 生成排查文档
结构
背景（2-3 句）+ 排查命令（3-5 条可执行 grep/curl）+ 结果汇总表（命中/文件/结论）。
红线
不写后端路径；每条命令可复制执行；涉及具体值须给「错误值 vs 正确值」。
04 写入 + 自检
产出
开发过程/{简短文件名}.md。
检查
无后端路径、命令可执行、有结果汇总表、背景≤5 行、参考标注「找不到忽略」。

关键信息

触发条件

  • 前端参数类错误（后端已确认 OK）
  • 想确认前端是否引用特定/废弃数据源
  • 想知道前端怎么调某接口
  • 任意前端确认类问题

排除边界

  • 后端 500 错误 → debug-backend-500
  • 前端 404 路由问题 → debug-frontend-404
  • 直接修改后端代码（本 Skill 只产排查文档）
  • 后端未经确认正确前不启用

产出物

  • 开发过程/{文件名}.md（给前端 agent 执行的排查文档）
  • 背景 + 3-5 条 grep/curl + 结果汇总表
