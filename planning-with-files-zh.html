执行全流程（启动 → 三文件 → 收尾）

01 启动前必执行（最高优先级）
动作
精读 .ai-workflow/planning/ 下 3 个核心文件（不存在则创建：STATUS / CHANGELOG / PROJECT_MAP）。
关卡
冲突校验——确认本次文件路径 / 功能与现有内容无冲突，禁止重复开发。
02 维护三文件工作流
文件
task_plan.md（目标/阶段/勾选/问题）、notes.md（研究/思路/上下文）、[deliverable].md（交付物）。
规则
只追加不覆盖、每阶段完成立即打勾、所有上下文入文件、关键决策前重读 task_plan。
03 开发过程强制规范
规范
全程中文；适配 macOS（不用 Linux 专属指令）；Python 遵循 PEP8、JS/TS 遵循 ESLint；Python3 先激活 venv。
04 完成后自动更新文档
动作
增量补充（不改原结构）：STATUS 更新进度百分比；CHANGELOG 按语义化版本（Added/Fixed/Changed）；PROJECT_MAP 树状补充新增文件与接口。

关键信息

触发条件

  • 复杂 / 多步骤任务或项目开发
  • 需要持久化进度追踪与状态管理
  • 需要冲突校验、可追溯的开发管理

排除边界

  • 一次性简单问答（无需持久化规划）
  • 不涉及文件修改的纯分析任务
  • 不需要状态管理的轻量操作
  • 项目缺少 STATUS/CHANGELOG/PROJECT_MAP 时不适用

产出物

  • .ai-workflow/planning/task_plan.md
  • .ai-workflow/planning/notes.md
  • .ai-workflow/planning/STATUS.md / CHANGELOG.md / PROJECT_MAP.md
