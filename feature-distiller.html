功能模块蒸馏器 feature-distiller

从任意项目里蒸馏单个功能 / 模块（如 RAG 检索链路、知识库、鉴权、支付）为自包含的可复用 HOW-TO 文档——含实现逻辑、数据流、棘手问题处理方法、避坑清单，全部带 file:line，从源头减少 AI 幻觉与重复踩坑。机械臂先行（scan_feature.py，纯标准库），AI 只填语义列。

执行全流程（5 步 SOP）

01 扫描（机械臂先行）
脚本
scripts/scan_feature.py --project <p> --feature <f> --output scan.json 多语言扫描，提取 matched_files / entry_points / definitions / imports / error_handling / guards / markers(TODO·FIXME·HACK·XXX) / bare_excepts。
红线
全部位置带 file:line，确定性数据零编造。
02 填语义列（实现逻辑 / 数据流）
动作
AI 基于 scan.json 描述「实现逻辑 / 数据流」，只描述 JSON 中存在的内容。
红线
无来源不写；不读取源码就不下结论。
03 棘手问题处理
提炼
从 error_handling（try/except/catch）与 guards（is None / empty 检查）提炼边界、异常、并发处理策略。
04 避坑清单
提炼
从 markers（原作者显式标记）与 bare_except / not_implemented（隐性雷区）提炼「不要做什么、为什么」。
05 组装文档
动作
套用 references/output-template.md，产出 <feature>-distilled.md。
产出
写入 <项目>/.ai-workflow/feature-distiller/，含 5 段：功能定位 / 实现逻辑与数据流 / 棘手问题处理 / 避坑清单 / 跨项目复用指引。
