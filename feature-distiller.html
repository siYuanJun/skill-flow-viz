研发层 · 框架契约蒸馏

框架开发契约结构化抽取引擎 feature-distiller · V2.0.0

输入目标项目源码，输出一份「AI 可直接解析、零额外讲解」的开发规范契约文件。AI 读取后即可严格遵循框架原生的目录、依赖、基类、权限、配置、编码约定进行开发，避免乱写路径、重写底层、破坏架构。机械臂先行（scan_framework.py，纯标准库），AI 只填语义列。本页为 V2 升级版，支持双模式蒸馏 + 全维度语义抽取 + 单文件契约 + 三级约束分级 + 自检门控。

执行全流程（5 步 SOP · 对应五层引擎）

01 模式确认（交互入口层 · 模式分流）
交互
启动后先确认：模式（①全框架全局蒸馏 / ②单业务模块蒸馏）× 范围（后端 FastAPI / 前端 Vue3 / 前后端全量）× 项目根目录绝对路径；模块模式还需目标业务模块名。信息不全不擅自开始蒸馏。
分流
全局模式产出底层总契约（一次蒸馏、永久复用）；模块模式只抽表结构 / 接口 / 页面 / 专属逻辑，自动继承全局规则、不重复冗余。
02 深度语义扫描（语义扫描层 · 机械臂先行）
脚本
scripts/scan_framework.py --project <p> --mode global|module --scope backend|frontend|full [--module <m>] --output contract-scan.json
后端
用 ast 精确提取：依赖环境 / 目录契约(is_core 禁用标记) / 核心基类(Base·BaseCRUD·BaseModel) / 配置体系(Settings+env) / 数据库(ORM·Base·模型·Alembic) / 接口路由(FastAPI+Flask) / 中间件 / 工具库 / 编码约定 / 测试部署。
前端
用正则提取：Vue 版本 / 路由+守卫 / axios 封装·拦截器 / 权限指令 v-auth / Pinia store / vite·.env。
红线
全部位置带 file:line，确定性数据零编造；旧 scan_feature.py 仍保留，供「单功能 HOW-TO 蒸馏」兼容使用。
03 规则提炼与分级（知识加工层）
三步加工
①去重聚合（重复约定聚为全局通用规则）；②分级标注 🔴强制🟡推荐⚫禁止；③示例补充（继承 BaseCRUD、写接口、注册路由等最简可套用示例）。
红线
每条规则必须标级；AI 开发时严格遵守 🔴 强制项，绝不触碰 ⚫ 禁止项。
04 结构化输出（产物输出层 · 单文件契约）
模板
套用 references/contract-template.md，固定 10 节：文档头 → 依赖清单 → 目录契约 → 核心基类 → 配置体系 → 数据库规范 → 接口/页面标准 → 工具与通用能力 → 编码约定 → 禁止操作清单 + 最简开发示例。模块版仅含表定义 / 接口 / 页面 / 专属权限。
产出
<框架名>-开发契约.md 落地 <项目>/.ai-workflow/feature-distiller/，单文件、层级固定、标签明确，AI 可快速检索定位。
05 自检校验（自检校验层 · 质量兜底）
必检
后端：依赖 / 目录 / Base模型 / BaseCRUD / JWT鉴权 / 配置 / 迁移工具 / 统一响应；前端：目录 / 路由 / 请求封装 / 权限指令 / 环境配置。
兜底
缺失项自动回溯补充扫描，不输出残缺契约。

关键信息

双模式蒸馏

① 全局 全框架总契约 ② 模块 业务补充契约 范围：后端/前端/全量

模块契约自动继承全局规则，不重复冗余；一次全局蒸馏，后续所有模块开发直接复用。

三级约束分级

🔴 强制约束 🟡 推荐实践 ⚫ 禁止操作

每条规则标注优先级，核心底层目录（如 core/）标 ⚫ 禁止修改。

触发条件

  • 接手新/未知框架，想让 AI 踩在真实源码上开发
  • 希望统一多智能体（如 DevSquad）的开发规范底座
  • 固化某框架的目录 / 基类 / 权限 / 配置 / 编码约定

排除边界（Out of scope）

  • 整库蒸馏 → repo2skill / repo-to-skill
  • 项目价值简历 → project-cv-generator
  • 代码调用图谱 → graphify
  • 单功能 HOW-TO（旧能力）→ 仍可用 scan_feature.py

产出物

  • <框架名>-开发契约.md（单文件 10 节，全部 file:line 溯源）
  • 可作 DevSquad 多智能体知识库，统一开发标准
