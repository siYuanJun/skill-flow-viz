API Test Harness

强制真实执行的接口机械臂测试：先从源码 grep 提取真实参数，再生成可直接执行的 Python 脚本 + curl 命令，杜绝 AI 编造测试结果。支持 Python/Java/Go/Node.js。四道防幻觉红线 + 五步强制流程。

从开始到完成，每一步做什么、产出什么一目了然。◆ 菱形为人工审批闸口（Gate）。下方时间轴为逐步详情。

执行全流程（五步不可跳步）

00 识别技术栈
动作
从代码自动判断路由框架（Flask/FastAPI / Spring Boot / Go Gin / Express / Nuxt），一句话告知用户。
01 机械臂提取（核心防幻觉）
动作
grep 源码提取方法/路径/参数/类型/必填，每条标注 文件:行号；提取后表格展示请用户确认。
红线
不得编造参数——所有参数必须来自源码且标注来源。
02 生成 Python 测试脚本
动作
单文件，标准库 + requests，可直接 python test_xxx.py 运行；含超时、连接失败友好提示。
红线
必须真实执行（输出真实 HTTP 响应），不得写「模拟返回」。
03 生成 curl 命令文档
动作
每个接口必附可独立执行的 curl（正常 / 缺参校验 / 带 Token 场景）。
红线
curl 必产——不让用户脱离 Python 也能验证。
04 保存到 .ai-workflow/api-test/
产出
test_api.py + curl_commands.md + extraction_report.md（参数来源表）+ 运行后 test_result.json。

  • 多技术栈：Python/Java/Go/Node.js
