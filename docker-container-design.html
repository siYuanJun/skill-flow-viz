执行全流程

01 分析项目
输入
技术栈（Node/Java/Python/PHP/Go）、根目录、端口、启动命令、构建产物、部署环境、资源预估。
02 编写 Dockerfile（纯净镜像）
红线
绝不写死 ENV APP_ENV；所有 FROM 强制走国产镜像源（禁止直连 Docker Hub）。
03 docker-compose.yml（生命周期全管）
覆盖
build/networks/deploy.resources/healthcheck/logging/restart/volumes；生产环境不走 ports 映射。
04 增量更新脚本（≤30 行）
动作
cli/update_app.sh 仅 docker cp + docker restart，不重复实现 compose 功能。
05 导出脚本 + 环境切换验证
动作
scripts/export_images.sh 按服务名导出 tar（离线/内网部署）；APP_ENV=production docker-compose up -d 验证同镜像多环境。

关键信息

触发条件

  • 首次容器化（Dockerfile + compose）或审查已有方案
  • 离线部署 / 容器网络拓扑设计
  • compose 与增量脚本职责划分
  • 国内镜像拉取超时、时区、数据持久化

排除边界

  • Kubernetes / Helm
  • CI/CD 流水线、云平台 IaC
  • docker CLI 调试命令、容器安全扫描

产出物

  • Dockerfile（零环境硬编码）
  • docker-compose.yml（网络+资源+健康检查）
  • scripts/export_images.sh + cli/update_app.sh
