执行全流程（Planning → Gate → Execution）

init 初始化编排器
命令
loop_orchestrator.py init "需求" /path/to/project
动作
载入项目，进入 Planning Loop（Stage 1-5），仅产出文档不写代码。
01 需求澄清（机械臂 build_index / find_similar）
动作
构建项目结构索引，搜索相似已有模块，产出需求文档。
输出
项目索引 JSON + 需求文档
02 方案设计（find_similar / scan_imports）
动作
基于事实对比多套实现方案。
输出
多方案对比
03 影响评估（find_callers / scan_imports / scan_schema）
动作
查找符号调用位置、双向依赖图、数据库表结构——核心防幻觉环节。
输出
影响分析报告
04 详细设计
输出
设计文档
05 分步指导
输出
分步计划
⏸ 人工审批 GATE
动作
Planning 只出文档；审批通过后 advance --approve-execution 才进入 Execution。
06 逐步实现（Execution）
输出
代码
07 测试文档（Execution）
输出
测试报告

关键信息

触发条件

  • 已有代码基础上新增 / 修改功能（需影响评估）
  • 大型项目做全方位审计分析
  • 需产出可观测、可审批的开发文档

排除边界

  • Greenfield 新项目（无存量代码，无需影响评估）
  • 一次性代码片段
  • 纯 bug 修复（已有 debug 类 Skill）
  • UEP / AI Dev Workflow 已覆盖的工程流

产出物

  • .ai-workflow/meta-feature-dev/STATE.json
  • 每阶段目录含脚本 JSON + AI 分析文档
  • 影响分析报告 / 设计文档 / 分步计划 / 测试报告
